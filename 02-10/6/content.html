Chat

You're headed to San Francisco for your awesome summer internship and in a stroke of luck ended up on the same plane as several of your friends! Unfortunately, you're all in different seats and internet on planes is expensive (not to mention crappy!). But the plane does provide a chat system so that you can chat with all of your friends during the flight.

On the chat system you can send messages, receive messages, and delete conversations. Conversations are ordered on your screen based on the most recent action that wasn't a delete (so a send is equivalent to a receive, and the most recent conversation that has had a send or receive will appear at the top of your screen).

Much to your annoyance, you can only delete conversations at the top of the screen, so each time a delete action is performed the most recently interacted with conversation is deleted.

Given all of the actions that occurred in order of their occurrence (with the oldest action first), can you keep track of which conversation is being deleted?

WARNING: This problem is both input and output heavy. You probably want to use fast IO in your language of choice to avoid timing out.

Input

The input will begin with a single integer T, the number of test cases.

T test cases follow. Each test case begins with an integer I, the number of interactions you have with the messaging system. I interactions follow. Each interaction will take one of the following forms:

  • You receive a message from person named NAME: RECEIVE NAME
  • You send a message to person named NAME (and read any unread messages from NAME): SEND NAME
  • You delete the conversation at the top of the screen: DELETE

Interactions are listed in the order that they occurred and you may assume that no interactions occurred at the same time.

You may assume that names with different capitalizations are distinct names.

Output

Each time the DELETE command appears, print out the name of the person from the conversation that was deleted.

Each printed name should be on its own line. Print a single blank line after each test case. If DELETE is called and there are no conversations to delete, print "NONE".

Constraints

1 <= T <= 10
0 <= I <= 110000
NAME will be at most 10 characters long and will contain only alphabetic characters.

Sample Input

1
4
SEND Ben
RECEIVE April
RECEIVE Jenny
DELETE

Sample Output

Jenny

Explanation

After the initial actions your list would be: Jenny, April, Ben. Jenny is at the front, so her name is deleted.
